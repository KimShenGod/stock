# 策略回测

本项目提供高性能向量化回测引擎，支持策略组合回测和收益分析。

## 回测系统架构

```
backtest/
├── __init__.py              # 模块入口，导出所有类
├── local_data_loader.py     # 本地数据加载器
├── signal_calculator.py     # 信号预计算模块
├── streaming_strategy.py    # 流式策略配置
├── vectorized_backtest_engine.py  # 向量化回测引擎
└── batch_backtest_engine.py       # 增量批处理引擎
```

## 快速开始

### 命令行方式

```bash
# 全流程（预计算信号 + 回测）
python run_vectorized_backtest.py --mode full --start 20240101 --end 20241231

# 仅预计算信号
python run_vectorized_backtest.py --mode precompute --start 20240101 --end 20241231

# 仅运行回测
python run_vectorized_backtest.py --mode backtest --strategy-combo default
```

### Python API方式

```python
from backtest import quick_backtest

# 快速回测
result = quick_backtest(
    start_date='20240101',
    end_date='20241231',
    strategy_names=['周线MACD区间', '小市值'],
    max_positions=3,
    initial_capital=400000,
    stop_loss_pct=-0.08,
    take_profit_pct=0.30,
    max_hold_days=30,
)

# 查看结果
print(f"总收益: {result['total_return_pct']:.2f}%")
print(f"夏普比率: {result['sharpe_ratio']:.2f}")
print(f"最大回撤: {result['max_drawdown_pct']:.2f}%")
```

## 回测引擎选择

### 向量化引擎（推荐）

特点：
- 全量矩阵一次性加载
- NumPy/Pandas向量化运算
- 可选Numba JIT加速
- 10-50倍速度提升

适用场景：内存充足（建议8GB+）

### 增量批处理引擎

特点：
- 分批处理，释放内存
- 支持断点续传
- 适合大规模数据

适用场景：内存受限或股票数量极多

```bash
# 强制使用增量引擎
python run_vectorized_backtest.py --mode full --no-vectorized
```

## 回测配置

### config.yml配置

```yaml
backtest:
  initial_capital: 400000   # 初始资金
  max_positions: 3          # 最大持仓数
  stop_loss: 0.08           # 止损比例 8%
  take_profit: 0.30         # 止盈比例 30%
  max_hold_days: 30         # 最大持仓天数
  commission_rate: 0.0003   # 佣金率 0.03%
  stamp_tax_rate: 0.001     # 印花税率 0.1%
  signal_dir: "./backtest_results/signals"
  output_dir: "./backtest_results"
  batch_days: 30            # 批处理天数
```

### 策略组合配置

```yaml
strategies:
  default: ["周线MACD区间", "小市值"]
  combo_1: ["前日涨停", "小市值"]
  combo_2: ["高开涨停", "换手率"]
  combo_3: ["今日涨停", "小市值"]
```

策略组合为AND关系，需同时满足所有条件才触发买入。

## 信号预计算

### 工作原理

信号预计算将所有策略的买卖信号提前计算并保存：

1. 加载所有股票历史数据
2. 执行所有策略函数
3. 生成买入信号矩阵
4. 保存为pickle批次文件

### 信号文件结构

```
backtest_results/signals/
├── signal_config.pkl           # 信号配置元数据
├── signals_batch_0000_of_0091.pkl  # 信号批次文件
├── signals_batch_0001_of_0091.pkl
└── ...
```

### 预计算参数

```python
from backtest import SignalCalculator, SignalConfig

calculator = SignalCalculator(output_dir='./signals')
config = SignalConfig(
    start_date='20240101',
    end_date='20241231',
    lookback_days=120,    # 策略计算所需的前置数据
    batch_size=100,       # 每批次股票数
)

calculator.calculate_and_save_batch(
    symbols=stock_list,
    config=config,
    strategy_names=['周线MACD区间', '小市值'],
    batch_size=50,
)
```

## 回测逻辑说明

### 买入逻辑

1. 检查买入信号（策略组合AND关系）
2. 检查可用持仓槽位
3. 等权重分配资金
4. 按开盘价买入（可配置）
5. 计算佣金成本

### 卖出逻辑

卖出触发条件（任一满足即卖出）：

| 条件 | 说明 |
|-----|-----|
| signal | 策略卖出信号触发 |
| stop_loss | 亏损超过止损比例 |
| stop_profit | 盈利超过止盈比例 |
| max_hold | 持仓天数达到上限 |

卖出执行：
- 按开盘价卖出（可配置）
- 计算佣金+印花税成本
- 记录盈亏

## 高级用法

### 自定义引擎配置

```python
from backtest import VectorizedBacktestEngine, VectorizedBacktestConfig

config = VectorizedBacktestConfig(
    initial_capital=400000,
    max_positions=3,
    stop_loss_pct=-0.08,
    stop_profit_pct=0.30,
    max_hold_days=30,
    buy_signal_type='周线MACD区间_buy',
    buy_time='open',
    sell_time='open',
    use_numba=True,
)

engine = VectorizedBacktestEngine(
    signal_dir='./signals',
    data_dir='./TDXdata/pickle',
    config=config,
)

result = engine.run('20240101', '20241231')
engine.save_result()
```

### 批处理引擎配置

```python
from backtest import IncrementalBacktestEngine, BatchBacktestConfig

config = BatchBacktestConfig(
    initial_capital=400000,
    batch_duration_days=30,
    overlap_days=5,
    signal_dir='./signals',
    output_dir='./results',
)

engine = IncrementalBacktestEngine(config)
result = engine.run('20240101', '20241231')
```
